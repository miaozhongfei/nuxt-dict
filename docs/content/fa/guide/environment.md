**هدف**: نصب Node.js و pnpm روی کامپیوتر، سپس ایجاد یک پروژه Nuxt قابل اجرا. پس از این فصل یک پروژه خالی Nuxt قابل دسترس در `http://localhost:3000` خواهید داشت.

## چه زمانی باید این فصل را بخوانید؟

- اولین بار است که با توسعه frontend آشنا می‌شوید و هیچ ابزار frontend روی کامپیوتر ندارید
- کامپیوتر شما مجدداً نصب شده و باید محیط را دوباره راه‌اندازی کنید
- Node.js را نصب کرده‌اید اما با pnpm آشنا نیستید

## مرحله ۱: نصب Node.js

Node.js محیط اجرای پایه Nuxt است. Nuxt 4 به **Node.js >= 22.0.0** نیاز دارد.

### ۱.۱ دانلود بسته نصب

وب‌سایت رسمی Node.js را باز کنید: [https://nodejs.org](https://nodejs.org)

دو نسخه می‌بینید:
- **LTS** (پشتیبانی بلندمدت): توصیه شده برای اکثر کاربران، پایدار و قابل اعتماد
- **Current** (آخرین نسخه): شامل آخرین ویژگی‌ها، اما ممکن است باگ داشته باشد

**نسخه LTS را انتخاب و دانلود کنید.**

### ۱.۲ نصب

پس از دانلود، فایل نصب را دوبار کلیک کرده و "Next" را ادامه دهید.

در حین نصب گزینه‌ای به نام **"Automatically install the necessary tools"** وجود دارد — آن را تیک بزنید (به طور خودکار ابزارهای کامپایل مورد نیاز را نصب می‌کند).

### ۱.۳ تأیید نصب

پس از نصب، ترمینال را باز کرده و دستور زیر را اجرا کنید:

```bash
node --version
```

باید خروجی مشابه زیر ببینید:

```
v22.12.0
```

اگر شماره نسخه >= `22.0.0` باشد، نصب موفقیت‌آمیز بوده است.

## مرحله ۲: نصب pnpm

pnpm یک مدیر بسته JavaScript است. می‌توانید آن را مانند یک "فروشگاه برنامه" تصور کنید — دیگران کد خود را به صورت بسته منتشر می‌کنند و شما با pnpm آن‌ها را دانلود و نصب می‌کنید.

پروژه Nuxt Dict اجباری از pnpm استفاده می‌کند و نمی‌توان از npm یا yarn استفاده کرد.

### ۲.۱ نصب pnpm

در ترمینال وارد کنید:

```bash
npm install -g pnpm
```

معنی این دستور:
- `npm install` = نصب یک بسته با npm
- `-g` (global) = نصب سراسری، پس از آن در هر دایرکتوری قابل استفاده است
- `pnpm` = نام بسته مورد نظر

### ۲.۲ تأیید نصب

```bash
pnpm --version
```

خروجی باید >= `10.22.0` باشد.

### ۲.۳ پیکربندی منبع آینه (اختیاری، توصیه شده)

محیط شبکه داخل ایران ممکن است دسترسی به مخزن npm خارجی را کند کند:

```bash
pnpm config set registry https://registry.npmmirror.com/
```

## مرحله ۳: ایجاد پروژه Nuxt

### ۳.۱ مقداردهی اولیه پروژه

ترمینال را باز کرده، به دایرکتوری مورد نظر برای کد پروژه بروید، سپس اجرا کنید:

```bash
npx nuxi init my-app
```

این دستور:
- `npx nuxi` = دانلود و اجرای موقت `nuxi` (ابزار ایجاد پروژه Nuxt)
- `init my-app` = ایجاد یک پروژه به نام `my-app`

در حین اجرا، چند سوال می‌پرسد:
- **Which package manager would you like to use?** ← **pnpm** را انتخاب کنید
- **Initialize git repository?** ← **Yes** را انتخاب کنید (توصیه می‌شود)

### ۳.۲ ورود به دایرکتوری پروژه

```bash
cd my-app
```

### ۳.۳ نصب وابستگی‌ها

```bash
pnpm install
```

### ۳.۴ راه‌اندازی پروژه

```bash
pnpm dev
```

باید خروجی مشابه زیر ببینید:

```
Nuxt 4.x.x

> Local:    http://localhost:3000/
```

`http://localhost:3000/` را در مرورگر باز کنید. اگر صفحه خوش‌آمدگویی Nuxt را دیدید، پروژه با موفقیت ایجاد شده است!

### ۳.۵ توقف پروژه

`Ctrl + C` را در ترمینال فشار دهید.

## آنچه در این فصل آموختید

- [ ] نصب Node.js (نسخه >= 22) روی کامپیوتر
- [ ] نصب مدیر بسته pnpm
- [ ] ایجاد پروژه Nuxt با `npx nuxi init`
- [ ] راه‌اندازی سرور توسعه با `pnpm dev`
- [ ] بازدید `http://localhost:3000` در مرورگر

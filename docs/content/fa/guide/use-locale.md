**هدف**: یادگیری دریافت و تغییر زبان فعلی، و درک نحوه بازنشانی خودکار داده‌های دیکشنری پس از تغییر زبان.

## چه زمانی به این ویژگی نیاز دارید؟

- برنامه شما باید از تغییر زبان چینی/انگلیسی پشتیبانی کند
- پس از تغییر زبان، متن گزینه‌های تمام dropdownها باید به زبان مقصد تغییر کند
- به یک دکمه تغییر زبان نیاز دارید

## امضای کامل

```ts
useLocale(): {
  locale: Ref<string>
  setLocale: (newLocale: string) => void
  locales: string[]
}
```

## مقادیر بازگشتی

| ویژگی       | نوع                           | توضیح                                                               |
| ----------- | ----------------------------- | ------------------------------------------------------------------- |
| `locale`    | `Ref<string>`                 | کد زبان فعلی، مثلاً `'zh-CN'`، `'en-US'`                            |
| `setLocale` | `(newLocale: string) => void` | تغییر زبان، همزمان Cookie را به‌روزرسانی و کش دیکشنری را پاک می‌کند |
| `locales`   | `string[]`                    | لیست زبان‌های پشتیبانی شده                                          |

## مثال پایه: دکمه تغییر زبان

```vue
<template>
  <div>
    <p>زبان فعلی: {{ locale }}</p>
    <button
      @click="setLocale('zh-CN')"
      :style="{ fontWeight: locale === 'zh-CN' ? 'bold' : 'normal' }"
    >
      中文
    </button>
    <button
      @click="setLocale('en-US')"
      :style="{ fontWeight: locale === 'en-US' ? 'bold' : 'normal' }"
    >
      English
    </button>
  </div>
</template>

<script setup lang="ts">
const { locale, setLocale } = useLocale();
</script>
```

## مکانیزم تشخیص زبان

ماژول در زمان راه‌اندازی بر اساس `locale.source` پیکربندی شده، زبان فعلی را تشخیص می‌دهد:

- **`'cookie'` (پیش‌فرض)** — از cookie مشخص شده می‌خواند. به طور پیش‌فرض cookie با نام `i18n_redirected` خوانده می‌شود
- **`'header'`** — از هدر درخواست HTTP می‌خواند (فقط در رندر سمت سرور)
- **`'query'`** — از پارامتر URL می‌خواند. به طور پیش‌فرض پارامتر `lang` خوانده می‌شود

### زبان fallback

اگر هیچ‌یک از منابع بالا زبانی را تشخیص ندهند، از `locale.default` استفاده می‌شود:

```ts [nuxt.config.ts]
dict: {
  locale: {
    default: 'zh-CN',  // زبان پیش‌فرض
  }
}
```

## پس از تغییر زبان چه اتفاقی می‌افتد؟

فراخوانی `setLocale('en-US')`:

1. مقدار `locale` به `'en-US'` به‌روزرسانی می‌شود
2. اگر `source: 'cookie'` پیکربندی شده باشد، Cookie همگام‌سازی می‌شود
3. **تمام کش حافظه دیکشنری پاک می‌شود** (زیرا داده‌های دیکشنری برای زبان‌های مختلف متفاوت است)
4. **تمام درخواست‌های شبکه در حال انتظار لغو می‌شوند**
5. کامپوننت‌هایی که از `useDict` / `useDict` / `useDictTree` استفاده می‌کنند، به طور خودکار داده‌های دیکشنری زبان جدید را **دوباره بارگذاری** می‌کنند

> **یعنی نیازی به بازنشانی دستی دیکشنری نیست!** پس از تغییر زبان، کامپوننت‌ها به طور خودکار تشخیص می‌دهند که داده‌ها تغییر کرده‌اند (`data` به `null` تبدیل می‌شود) و درخواست جدید ارسال می‌کنند.

## نکات

> پس از تغییر زبان، کامپوننت‌هایی که از `useDict` / `useDictTree` / `useDict` استفاده می‌کنند **به طور خودکار دوباره بارگذاری می‌شوند** — نیازی به بازنشانی دستی یا نوشتن کد `watch` اضافی نیست.

> اگر از `@nuxtjs/i18n` استفاده می‌کنید، هر دو ماژول از طریق cookie مشترک (پیش‌فرض `i18n_redirected`) به طور خودکار همگام می‌شوند. [مشاهده جزئیات](/integration/i18n).

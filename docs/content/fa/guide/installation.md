# نصب ماژول

**هدف**: نصب ماژول Nuxt Dict در پروژه Nuxt ایجاد شده در فصل قبل و تأیید موفقیت‌آمیز بودن نصب.

## مرحله ۱: نصب بسته وابستگی

در دایرکتوری ریشه پروژه (`my-app`) ترمینال را باز کرده و اجرا کنید:

```bash
pnpm add @lacqjs/nuxt-dict
```

این دستور دو کار انجام می‌دهد:
1. بسته `@lacqjs/nuxt-dict` را در `node_modules/` دانلود می‌کند
2. این وابستگی را در `dependencies` فایل `package.json` ثبت می‌کند

> **تفاوت `dependencies` و `devDependencies` چیست؟**
> - `dependencies` = بسته‌های مورد نیاز در زمان اجرا (پروژه شما برای کار کردن به آن‌ها نیاز دارد)
> - `devDependencies` = بسته‌های مورد نیاز در زمان توسعه (مانند ابزارهای بررسی کد، ابزارهای تست، در محیط production لازم نیستند)
>
> Nuxt Dict یک وابستگی زمان اجراست، بنابراین از `pnpm add` (بدون `-D`) استفاده می‌شود.

## مرحله ۲: ثبت ماژول

فایل `nuxt.config.ts` را در ریشه پروژه باز کنید. `'@lacqjs/nuxt-dict'` را به آرایه `modules` اضافه کنید:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: [
    '@lacqjs/nuxt-dict'
  ],
})
```

> **`modules` چیست؟** سیستم ماژول Nuxt به دیگران اجازه می‌دهد یک بسته کاربردی بنویسند، شما فقط آن را در modules ثبت می‌کنید و مستقیماً قابل استفاده است. مانند نصب + فعال‌سازی یک App روی گوشی.

فایل را ذخیره کنید. اگر `pnpm dev` در حال اجراست، Nuxt به طور خودکار تغییرات را تشخیص داده و مجدداً بارگذاری می‌کند.

## مرحله ۳: تأیید نصب

### ۳.۱ مشاهده لاگ‌های کنسول

سرور توسعه را مجدداً راه‌اندازی کنید:

```bash
pnpm dev
```

در ترمینال، باید لاگ‌هایی مشابه زیر ببینید:

```
[nuxt-dict] راه‌اندازی ماژول @lacqjs/nuxt-dict
[nuxt-dict] گزینه‌های ماژول: { enable: true, logLevel: 3, ... }
```

دیدن این لاگ‌ها یعنی ماژول با موفقیت ثبت و راه‌اندازی شده است.

### ۳.۲ تأیید پشتیبانی TypeScript

در فایل `pages/index.vue` خود وارد کنید:

```vue [pages/index.vue]
<script setup lang="ts">
const { data } = useDict('gender')
</script>
```

اگر ویرایشگر خطایی نشان نداد و با تایپ `useDict` پیشنهاد تکمیل خودکار ظاهر شد، یعنی اعلان‌های نوع TypeScript نیز با موفقیت بارگذاری شده‌اند.

> **چرا بدون پیکربندی `dict: {}` هم کار می‌کند؟** تمام گزینه‌های پیکربندی ماژول مقادیر پیش‌فرض دارند. بدون نوشتن `dict: {}` از پیکربندی پیش‌فرض استفاده می‌شود که برای اکثر سناریوهای ساده کافی است. فصل [پیکربندی کامل](/advanced/full-config) هر گزینه را با جزئیات توضیح می‌دهد.

این فصل تمام فیلدهای پیکربندی `dict` در `nuxt.config.ts` را به عنوان مرجع سریع فهرست می‌کند.

## enable

| ویژگی   | مقدار                                                              |
| ------- | ------------------------------------------------------------------ |
| نوع     | `boolean`                                                          |
| پیش‌فرض | `true`                                                             |
| توضیح   | فعال یا غیرفعال کردن ماژول دیکشنری. `false` ماژول بارگذاری نمی‌شود |

## logLevel

| ویژگی   | مقدار                                                               |
| ------- | ------------------------------------------------------------------- |
| نوع     | `number`                                                            |
| پیش‌فرض | `3`                                                                 |
| توضیح   | سطح لاگ. 0=ساکت, 1=خطا, 2=هشدار, 3=اطلاعات, 4=اشکال‌زدایی, 5=جزئیات |

| ویژگی   | مقدار                                                                   |
| ------- | ----------------------------------------------------------------------- |

| پیش‌فرض | `'/api'`                                                                |
| توضیح   | آدرس پایه API. پشتیبانی از URL مطلق (API خارجی) یا مسیر نسبی (API محلی) |

## stores

| ویژگی   | مقدار                                                                                         |
| ------- | --------------------------------------------------------------------------------------------- |
| نوع     | `Record<string, { baseURL?, dictEndpoint?, versionEndpoint?, adapter? }>`                     |
| پیش‌فرض | `{}`                                                                                          |
| توضیح   | نگاشت پیکربندی چند مخزنی. هر مخزن می‌تواند endpoint مستقل یا آداپتور سفارشی خود را داشته باشد |

فیلدهای هر مخزن:

- **`baseURL`**: آدرس پایه API، در صورت عدم تنظیم از `api.baseURL` سراسری به ارث می‌رسد
- **`dictEndpoint`**: مسیر لیست دیکشنری، در صورت عدم تنظیم از `api.dictEndpoint` سراسری به ارث می‌رسد
- **`versionEndpoint`**: مسیر نسخه، در صورت عدم تنظیم از `api.versionEndpoint` سراسری به ارث می‌رسد
- **`adapter`**: آداپتور سفارشی برای این مخزن (**به ارث نمی‌رسد**؛ در صورت عدم تنظیم، آداپتور REST به طور خودکار ایجاد می‌شود)

[مشاهده جزئیات](/advanced/multi-store).

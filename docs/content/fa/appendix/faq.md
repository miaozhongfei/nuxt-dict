# سوالات متداول

## صفحه کد اصلی را نشان می‌دهد نه متن ترجمه شده

دلایل احتمالی:
1. داده‌های دیکشنری هنوز بارگذاری نشده‌اند. مطمئن شوید از `v-if="data"` یا `v-if="!loading"` برای translate استفاده می‌کنید
2. نام نوع دیکشنری اشتباه نوشته شده. تأیید کنید نام نوع در `useDict('gender')` با داده‌های برگشتی API backend مطابقت دارد
3. آدرس API اشتباه پیکربندی شده. `dict.api.baseURL` و `dict.api.dictEndpoint` را بررسی کنید

## useDict داده null برمی‌گرداند

مقدار اولیه `data` برابر `null` است و پس از mount کامپوننت به صورت ناهمگام بارگذاری می‌شود. از `loading` و `error` برای تشخیص وضعیت فعلی استفاده کنید.

## چگونه دیکشنری را به صورت دستی بازنشانی کنیم؟

تابع `refresh()` بازگشتی از composable را فراخوانی کنید.

## پس از تغییر زبان دیکشنری تغییر نکرده است؟

`setLocale()` کش حافظه را پاک می‌کند. کامپوننت‌هایی که از `useDict` / `useDictTree` / `useDict` استفاده می‌کنند **به طور خودکار دوباره بارگذاری می‌شوند** — نیازی به بازنشانی دستی یا نوشتن کد `watch` اضافی نیست.

## چگونه با @nuxtjs/i18n استفاده کنیم؟

هر دو ماژول از طریق cookie مشترک (پیش‌فرض `i18n_redirected`) به طور خودکار همگام می‌شوند. داده‌های دیکشنری در بازنشانی صفحه یا تغییر زبان در زمان اجرا به طور خودکار به‌روزرسانی می‌شوند. [مشاهده جزئیات](/integration/i18n).

## چگونه به API backend موجود متصل شویم؟

از [آداپتور سفارشی](/advanced/custom-adapter) استفاده کنید و داده‌های برگشتی backend را در `api.adapter` به فرمت مورد انتظار ماژول تبدیل کنید.

## چگونه به چندین منبع دیکشنری backend متصل شویم؟

از [چند مخزنی](/advanced/multi-store) استفاده کنید و endpointهای API مستقل برای هر منبع داده در `dict.stores` پیکربندی کنید.

## پیکربندی prefetch کار نمی‌کند؟

پیش‌واکشی فقط در حالت SSR (رندر سمت سرور) کار می‌کند. حالت SPA نیازی به پیکربندی prefetch ندارد.

## خطای IndexedDB چه کنم؟

شکست نوشتن IndexedDB بر عملکرد تأثیر نمی‌گذارد، ماژول به صورت بی‌صدا مدیریت می‌کند. دلایل رایج:
- حالت مرور خصوصی مرورگر
- فضای ذخیره‌سازی ناکافی
- مرورگر از IndexedDB پشتیبانی نمی‌کند (بسیار نادر)

می‌توانید `cache.indexedDB.enabled: false` را برای غیرفعال کردن کش IndexedDB تنظیم کنید.

## داده‌های دیکشنری به‌روزرسانی شده اما صفحه همچنان قدیمی است؟

1. مطمئن شوید API شماره نسخه backend شماره نسخه جدیدی برگردانده است
2. `refresh()` را فراخوانی کنید یا منتظر بمانید تا اولین دسترسی به داده‌های دیکشنری آن مخزن، تشخیص نسخه را فعال کرده و کش را به طور خودکار پاک کند

## خطای "fetch is not defined" در سرور

ماژول از fetch بومی Node.js 18+ استفاده می‌کند. مطمئن شوید محیط اجرا Node.js >= 22 باشد (نیازمندی Nuxt 4).

## نوع کد عددی است اما translate(1) کد اصلی را برمی‌گرداند

ماژول در داخل به طور یکسان به رشته تبدیل کرده و مقایسه می‌کند، بنابراین `translate(1)` و `translate('1')` یکسان هستند. اگر باز هم مطابقت ندارد، بررسی کنید کد برگشتی backend واقعاً `1` باشد (نه `'1'` یا مقدار دیگر).

## $dict.translatePath کد اصلی را برمی‌گرداند

دلایل احتمالی:
1. نوع دیکشنری هدف، دیکشنری درختی نیست (فیلد `tree` خالی است)
2. داده‌های دیکشنری هنوز در کش حافظه بارگذاری نشده‌اند
3. کد مورد نظر در درخت وجود ندارد
